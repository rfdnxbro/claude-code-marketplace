# 記事レビュー

作成した記事の品質チェックを行い、改善提案を実施します。

## 前提条件

- `/bbl:create-article` で記事が作成済みであること
- `.bbl-context.yml` が存在し、`phase` が `article_created` であること

## 手順

### 1. コンテキストの読み込み

`.bbl-context.yml` を読み込み、対象記事のパスを取得。

**エラーハンドリング**:

- ファイルが存在しない場合: 「`.bbl-context.yml`が見つかりません。先に `/bbl:brainstorm [概念名]` を実行してください。」
- `phase`が`article_created`でない場合:
  - `outline_created`: 「記事がまだ作成されていません。`/bbl:create-article` を実行してください。」
  - `completed`: 「この記事は既にレビュー済みです。」
  - その他: 現在のphaseを表示し、ワークフローを案内
- `file_path`で指定されたファイルが存在しない場合: 「記事ファイルが見つかりません。`/bbl:create-article` を再実行してください。」

### 2. 記事の読み込み

`file_path` から記事を読み込む。

### 3. 品質チェック

`content-reviewer` エージェントを呼び出し、以下の観点でレビュー：

#### 構造チェック（20点）
- 7セクションすべてが含まれているか
- 各セクションに十分な内容があるか

#### 導入問題チェック（20点）
- 実務シナリオベースになっているか
- 読者が自分ごととして考えられるか

#### 内容チェック（30点）
- 概念の定義が明確か
- 専門用語に説明があるか
- 具体例が実在企業の事例か

#### 視覚化チェック（15点）
- Mermaid図が1つ以上あるか
- 図が概念を適切に表現しているか

#### 参考文献チェック（15点）
- 参考文献が3つ以上あるか
- 関連概念へのリンクがあるか

### 4. レビュー結果の報告

以下の形式で結果を報告：

```markdown
## レビュー結果

**総合スコア**: XX/100点

### 評価詳細

| カテゴリ | スコア | コメント |
|---------|--------|----------|
| 構造 | /20 | |
| 導入問題 | /20 | |
| 内容 | /30 | |
| 視覚化 | /15 | |
| 参考文献 | /15 | |

### 改善が必要な箇所

1. [改善点1]
2. [改善点2]

### 推奨アクション

- [ ] アクション1
- [ ] アクション2
```

### 5. 修正の実施（オプション）

ユーザーの承認を得た後、必要な修正を実施。

### 6. コンテキストの更新

レビュー完了後、`.bbl-context.yml` を更新：

```yaml
phase: completed
review_score: [スコア]
updated: [現在時刻]
```

### 7. 完了報告

```
レビューが完了しました。

スコア: XX/100点
ステータス: [公開可能 / 修正が必要]

記事パス: [file_path]
```

## スコア基準

- **90点以上**: 公開可能
- **70-89点**: 軽微な修正が必要
- **70点未満**: 大幅な修正が必要（`/bbl:create-article` の再実行を推奨）
