# ブレインストーミング

指定された概念について対話を通じて内容を深掘りし、記事のアウトラインを作成します。

## 手順

### 1. 概念名の確認

引数として渡された概念名: `$ARGUMENTS`

概念名が指定されていない場合は、ユーザーに確認してください。

### 2. カテゴリの特定

以下の4カテゴリ・8サブカテゴリから適切なものを選択：

**ヒト**
- 組織行動・リーダーシップ
- 人材マネジメント

**モノ**
- マーケティング
- 経営戦略

**カネ**
- アカウンティング
- ファイナンス

**思考・志**
- クリティカルシンキング
- ビジネス倫理

ユーザーと対話してカテゴリを確定してください。

### 3. 内容の深掘り

以下の観点で対話を進める：

1. **定義と範囲**: この概念は何を指すのか
2. **重要性**: なぜこの概念を学ぶ必要があるのか
3. **理論的背景**: どのような研究や理論があるか
4. **実務での活用**: どのような場面で使われるか
5. **よくある誤解**: この概念についての誤解は何か

### 4. アウトラインの作成

対話を通じて得られた情報をもとに、以下のアウトラインを作成：

```yaml
導入問題:
  シナリオ: [具体的な業務状況]
  問い: [読者に考えさせる質問]

考えるポイント:
  - ポイント1
  - ポイント2
  - ポイント3

解説:
  定義: [概念の定義]
  重要性: [なぜ重要か]

詳細:
  理論: [理論的背景]
  フレームワーク: [使用するフレームワーク]
  図解: [Mermaid図の概要]

具体例:
  企業: [取り上げる企業]
  事例: [事例の概要]

関連概念:
  - 関連概念1
  - 関連概念2

参考文献候補:
  - 書籍や論文の候補
```

### 5. コンテキストの保存

アウトラインが確定したら、`.bbl-context.yml`を作成：

```yaml
concept: [概念名]
category: [ヒト/モノ/カネ/思考・志]
subcategory: [サブカテゴリ名]
outline: |
  [作成したアウトライン]
file_path: docs/[カテゴリ]/[サブカテゴリ]/[概念名].md
phase: brainstorm
updated: [現在時刻]
```

### 6. 次のステップの案内

ブレインストーミング完了後、以下を案内：

```
ブレインストーミングが完了しました。

次のステップ:
  /bbl:create-article を実行して記事を作成してください。
```
