# ビジネス基礎知識 コンテンツガイド

## 記事テンプレート

すべての記事は以下の7セクション構成に従う。

```markdown
# [概念名]

## 導入問題

[実務シナリオベースの問題を提示]
- 読者が自分ごととして考えられる具体的な状況
- 正解が一つではない、考えさせる問い

## 考えるポイント

[問題を考える際の観点を3-5点で提示]
- 何に着目すべきか
- どのような視点で分析すべきか

## 解説

[概念の基本的な説明]
- 定義と背景
- なぜこの概念が重要なのか
- 導入問題への適用

## 詳細

[理論的背景、フレームワーク]
- 学術的な裏付け
- フレームワークや理論の詳細
- Mermaid図による視覚化

## 具体例

[実際のビジネス事例]
- 成功事例・失敗事例
- 日本企業の事例を優先
- 具体的な数値や結果を含める

## 関連概念

[他の概念へのリンク]
- 同カテゴリ内の関連概念
- 他カテゴリとの関連性

## 参考文献

[書籍、論文、信頼性の高いWebソース]
- 学術論文や書籍を優先
- URLは信頼性の高いソースのみ
```

## Mermaid図の活用

理論やフレームワークの視覚化にMermaid図を積極的に活用する。

### 推奨する図の種類

| 図の種類 | 用途 |
|---------|------|
| flowchart | プロセス、意思決定フロー |
| graph | 関係性、構造 |
| sequenceDiagram | 時系列の相互作用 |
| mindmap | 概念の階層構造 |
| quadrantChart | 2軸での分類・位置づけ |

### 図の作成ガイドライン

- シンプルさを優先（ノード数は10以下推奨）
- 日本語ラベルを使用
- 色分けで意味を明確に

## 導入問題の作成ポイント

良い導入問題の条件：

1. **実務シナリオベース**: 抽象的な問いではなく、具体的な業務状況を設定
2. **自分ごと化**: 読者が当事者として考えられる設定
3. **正解が一つではない**: 複数の観点から検討できる問い
4. **概念の必要性を実感**: この概念を知らないと困る状況

### 例

```markdown
## 導入問題

あなたは新規事業開発チームのリーダーです。チームメンバーから
「このアイデアはうまくいかないと思うけど、言い出しにくい」
という声を個別に聞きました。

ミーティングでは活発な議論が行われているように見えますが、
実際には一部のメンバーの意見だけが採用されている状況です。

このチームの課題は何でしょうか？どのように改善すべきでしょうか？
```

## 品質チェックリスト

記事作成後、以下を確認：

- [ ] 7セクションすべてが含まれている
- [ ] 導入問題が実務シナリオベースになっている
- [ ] Mermaid図が1つ以上含まれている
- [ ] 具体例に実在の企業事例がある
- [ ] 参考文献が3つ以上ある
- [ ] 関連概念へのリンクがある
- [ ] 誤字脱字がない
- [ ] 専門用語には説明が添えられている

## ファイル配置

記事ファイルは以下のパスに配置：

```
docs/<カテゴリ>/<サブカテゴリ>/<概念名>.md
```

例:
- `docs/ヒト/組織行動・リーダーシップ/心理的安全性.md`
- `docs/カネ/ファイナンス/NPV.md`
- `docs/モノ/マーケティング/STP分析.md`
