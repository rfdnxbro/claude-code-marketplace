# ドキュメント編集プロセス

## 原則: 1項目ずつ確認しながら編集する

ドキュメントのブラッシュアップ時は、一度に全文を書き換えず、改善ポイントを1項目ずつユーザーに提示して確認を取りながら進める。

### なぜ

- 書き手の意図を損なうリスクを最小化する
- 複数の変更が混ざると差分レビューが困難になる
- ユーザーが各判断に関与でき、納得感のある仕上がりになる

### フロー

1. 全文を読み、改善ポイントをリストアップする
2. 各ポイントについてAskUserQuestionで候補を提示し、選択してもらう
3. 選択されたものだけEditで反映する
4. 全ポイント完了後、通しで最終チェックを行う

## 原則: 実データをベースに書く

参照先（Notionページ、ADR、外部URL等）がある場合は、必ずfetchして実際の内容を読んだ上でサマリや肩付けを作成する。

### なぜ

- 推測や捏造で書くと、事実と異なる内容が公式ドキュメントに載るリスクがある
- 参照先の情報量・ニュアンスを把握した上で、適切な要約度合いを判断できる

### やること

- Notion mention-pageのリンク先をfetchする
- 外部URLがあれば内容を確認する
- サマリ候補を複数提示し、ユーザーに選ばせる

### やらないこと

- 参照先を読まずに一般論で肩付けを書く
- 「〜と思われます」「〜の可能性があります」等の推測表現で補足を捏造する
