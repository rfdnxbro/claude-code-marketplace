# コード生成

あなたはソフトウェアエンジニアとして、ドメインモデルに基づいてコードを生成します。

## ワークフロー

### 1. 計画の作成

`aidlc-docs/<bolt>/plans/code_gen_plan.md` に計画を作成：

```markdown
# コード生成計画

## 対象コンポーネント
[コンポーネント名]

## ステップ
- [ ] ステップ1: ドメインモデルの確認
- [ ] ステップ2: エンティティ・値オブジェクトの実装（要確認）
- [ ] ステップ3: リポジトリ・ファクトリの実装
- [ ] ステップ4: サービス層の実装
- [ ] ステップ5: ユニットテストの作成
- [ ] ステップ6: テスト実行と修正
```

### 2. ドメインモデル参照

`aidlc-docs/<bolt>/design-artifacts/domain/` のドメインモデルを参照。

### 3. コード生成

ドメインモデルに基づいてコードを生成：

- エンティティ、値オブジェクト、集約の実装
- リポジトリとファクトリの実装
- サービス層の実装
- ユニットテストの作成

### 4. テスト実行

すべてのテストを実行し、結果を報告。失敗した場合は修正案を提示。

### 5. current.ymlの更新

完了後、current.ymlを更新：
- `step: architecture`
- `updated: <現在時刻>`
