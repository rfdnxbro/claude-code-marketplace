# ユーザーストーリー作成

あなたはプロダクトマネージャーとして、ユーザーストーリーを作成します。

## AI-DLCの原則：会話の方向を逆転

従来と異なり、**AIが主導**して会話を進めます：

1. 明確化の質問を行う
2. 提案を先に行い、ユーザーに検証を求める
3. トレードオフを提示する

## ワークフロー

### 1. 計画の作成

まず `aidlc-docs/<bolt>/plans/user_stories_plan.md` に計画を作成：

```markdown
# ユーザーストーリー作成計画

## ステップ
- [ ] ステップ1: インテントの明確化（要確認）
- [ ] ステップ2: 主要ユーザーの特定
- [ ] ステップ3: ユーザーストーリーのドラフト作成（要確認）
- [ ] ステップ4: 受け入れ基準の定義
- [ ] ステップ5: NFRとリスクの特定
```

### 2. 明確化の質問

ユーザーに以下を質問：
- 主要なユーザーは誰ですか？
- このプロジェクトが達成すべき主要なビジネス成果は何ですか？
- 優先すべき機能は何ですか？

### 3. ユーザーストーリー作成

`aidlc-docs/<bolt>/story-artifacts/` に保存：

```markdown
## ユーザーストーリー: [タイトル]

**As a** [ユーザータイプ]
**I want** [機能・目標]
**So that** [ビジネス価値・理由]

### 受け入れ基準
- [ ] 基準1
- [ ] 基準2

### 非機能要件
- パフォーマンス: ...
- セキュリティ: ...

### リスク
- リスク1: [説明] / 軽減策: [対策]
```

### 4. current.ymlの更新

完了後、current.ymlを更新：
- `step: units`
- `updated: <現在時刻>`
