# PRレビューコメント対応

PR `$1` のbotによるレビューコメントを確認し、未対応の指摘事項を修正します。

## 手順

### 1. PR情報の取得

以下のコマンドでPRのレビューコメントを取得:

```bash
gh pr view $1 --json number,title,state,headRefName
gh api repos/{owner}/{repo}/pulls/$1/comments --jq '.[] | select(.user.type == "Bot") | {id, user: .user.login, body, path, line: .original_line, created_at}'
gh api repos/{owner}/{repo}/pulls/$1/reviews --jq '.[] | select(.user.type == "Bot") | {id, user: .user.login, state, body}'
```

**注意**: `{owner}/{repo}` は `gh repo view --json nameWithOwner -q .nameWithOwner` で取得すること。

### 2. 未対応コメントの特定

取得したbotコメントを分析し、以下に分類:

- **修正が必要**: コードの品質・バグ・セキュリティに関する妥当な指摘
- **対応不要**: 誤検知、スタイル上の好み、プロジェクト方針と合わない指摘
- **判断保留**: 修正の妥当性が判断できない指摘

### 3. コードの修正

「修正が必要」と判断した指摘について:

1. 該当ファイルを読み込む
2. 指摘内容に基づいてコードを修正する
3. 修正が他の箇所に影響しないか確認する

### 4. 修正内容をレビューコメントに返信

修正した各指摘に対して、元のレビューコメントへの返信として修正内容を記録する。
これにより、レビュー履歴上で「何をどう直したか」が追跡可能になる。

botのレビューコメントのIDを使い、以下のコマンドで返信:

```bash
gh api repos/{owner}/{repo}/pulls/$1/comments/{comment_id}/replies \
  -method POST -f body='対応内容'
```

返信本文は以下の形式:

```
✅ 対応しました

**修正内容**: （具体的に何をどう変更したかを簡潔に記述）
**対象ファイル**: `path/to/file`
**コミット**: （コミットハッシュ）
```

「対応不要」と判断した指摘にも返信し、理由を明記する:

```
⏭️ 対応不要と判断

**理由**: （対応不要と判断した具体的な理由）
```

### 5. 変更のコミットとプッシュ

修正がある場合:

```bash
git add <修正したファイル>
git commit -m "fix: botレビュー指摘対応"
git push
```

**注意**: コミットハッシュは手順4の返信に含めるため、コミット後に返信を送ること。

### 6. 結果報告

以下の形式で報告:

```
## レビュー対応状況

**PR**: #$1
**確認したbotコメント数**: X件

### 修正済み
- [bot名] ファイル:行 - 指摘内容の要約

### 対応不要（理由付き）
- [bot名] ファイル:行 - 指摘内容 → 不要理由

### 判断保留
- [bot名] ファイル:行 - 指摘内容

### ステータス
- [ ] 全指摘対応完了 → ループを停止してください
- [ ] 未対応あり → 次回ループで再確認
```

### 7. ループ終了判定

全ての指摘が「修正済み」または「対応不要」になった場合、「全指摘対応完了。ループを停止してください。」と報告し、ループのキャンセルを提案する。

**重要**: プッシュ後にCIが再実行され新たなコメントが付く可能性があるため、最低1回は再確認してから完了と判断すること。
